# GLASS v0.1.0 Windows GPU Alpha 1

This is the first Windows GPU alpha release for GLASS.

## Downloads

- `GLASS-v0.1.0-windows-gpu.1-Portable-win64-cpu.zip`
  - CPU-capable portable package.
- `GLASS-v0.1.0-windows-gpu.1-Portable-win64-cuda11.zip`
  - CUDA 11.8 build for older NVIDIA GPUs.
  - Compiled for compute capabilities 5.2, 6.1, 7.5, and 8.6.
- `GLASS-v0.1.0-windows-gpu.1-Portable-win64-cuda12.zip`
  - CUDA 12.9 build for Turing, Ampere, Ada, and Hopper-class GPUs.
  - Compiled for compute capabilities 7.5, 8.6, 8.9, and 9.0.
- `GLASS-v0.1.0-windows-gpu.1-Portable-win64-cuda13.zip`
  - CUDA 13.0 build for newer NVIDIA GPUs, including Blackwell-class targets.
  - Compiled for compute capabilities 8.6, 8.9, 9.0, and 12.0.
- `glass_stack-0.1.0-py3-none-any.whl`
- `glass_stack-0.1.0.tar.gz`

## Which Package Should I Use?

- If you do not have an NVIDIA GPU, use the CPU package.
- If your GPU is Maxwell, Pascal, Turing, or Ampere-era, start with the CUDA 11 package.
- If your GPU is Turing, Ampere, Ada, or Hopper-era, start with the CUDA 12 package.
- If your GPU is Blackwell-era or you have a very recent NVIDIA driver, try the CUDA 13 package.

## Quick Start

```powershell
.\glass-doctor.cmd
.\glass.cmd --help
.\glass.cmd synthetic --out runs\demo_synth --frames 8 --width 128 --height 128 --filter H --known-shift
.\glass.cmd audit --root runs\demo_synth --out runs\demo_audit --backend auto
```

`glass doctor` reports whether the native CUDA extension loaded successfully.
If the CUDA package does not load on a specific machine, use the CPU package or
try the adjacent CUDA version.

## Notes

- These are portable alpha packages intended for installation and runtime validation.
- The CUDA packages are built with static CUDA runtime linkage where supported.
- Users still need an NVIDIA display driver compatible with the selected CUDA runtime.
