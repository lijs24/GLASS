# GLASS v0.1.0 Windows GPU Alpha 8

This Windows GPU alpha improves CUDA package compatibility guidance and rebuilds
the portable CUDA packages with a wider native architecture matrix. It also
fixes the portable Windows runtime so the downloaded zip no longer depends on
the Python path from the build machine, and bundles the CUDA runtime DLL when a
toolkit build still needs it at load time.

## Downloads

- `GLASS-v0.1.0-windows-gpu.8-Portable-win64-cpu.zip`
  - CPU-capable portable package.
- `GLASS-v0.1.0-windows-gpu.8-Portable-win64-cuda11.zip`
  - CUDA 11.8 package.
  - Native targets: compute capabilities 5.0, 5.2, 6.0, 6.1, 7.0, 7.5, 8.0, and 8.6.
  - Newer GPUs can try this package as a PTX JIT compatibility fallback.
- `GLASS-v0.1.0-windows-gpu.8-Portable-win64-cuda12.zip`
  - CUDA 12.4 package.
  - Native targets: compute capabilities 7.5, 8.0, 8.6, 8.9, and 9.0.
  - Newer GPUs can try this package as a PTX JIT compatibility fallback.
- `GLASS-v0.1.0-windows-gpu.8-Portable-win64-cuda13.zip`
  - CUDA 13.0 package.
  - Native targets: compute capabilities 8.6, 8.9, 9.0, 10.0, and 12.0.
  - Preferred first choice for Blackwell-class cards such as RTX PRO 6000 Blackwell.
- `glass_stack-0.1.0-py3-none-any.whl`
- `glass_stack-0.1.0.tar.gz`

## Which Package Should I Use?

- If you do not have an NVIDIA GPU, use the CPU package.
- If you have RTX PRO 6000 Blackwell or another compute capability 12.0 card,
  start with the CUDA 13 package for native performance.
- If the preferred CUDA package does not load, try the next package down:
  `cuda13`, then `cuda12`, then `cuda11`, then CPU.
- New GPUs can run older CUDA packages through PTX forward JIT on a sufficiently
  new NVIDIA driver. First launch may be slower, and peak performance can be
  lower than the native package.

## Quick Start

```powershell
.\glass-doctor.cmd
.\glass.cmd --help
.\glass.cmd synthetic --out runs\demo_synth --frames 8 --width 128 --height 128 --filter H --known-shift
.\glass.cmd audit --root runs\demo_synth --out runs\demo_audit --backend auto
```

`glass doctor` reports the detected GPU, CUDA availability, and the Windows
package try order for the current machine.

## Notes

- These are portable alpha packages intended for installation and runtime validation.
- Users need a compatible NVIDIA display driver for CUDA packages.
- Users do not need the CUDA Toolkit, Visual Studio, CMake, or Ninja to run the
  portable packages.
